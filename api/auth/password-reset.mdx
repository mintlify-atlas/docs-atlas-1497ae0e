<Warning>
**Configuration Required**: This endpoint requires [AshJsonApi setup](/api/setup) before it will work. By default, Ash Coffee uses browser-based authentication.
</Warning>

The password reset flow consists of two steps: requesting a reset token and using that token to set a new password.

## Request Password Reset Token

### Endpoint

```
POST /api/json/password-reset/request
```

Send a password reset email to the user with a secure token.

### Request

#### Headers

<ParamField header="Content-Type" type="string" required>
  Must be `application/vnd.api+json`
</ParamField>

#### Body Parameters

<ParamField body="data" type="object" required>
  The request data object
  
  <Expandable title="properties">
    <ParamField body="type" type="string" required>
      Must be `"password_reset"`
    </ParamField>
    
    <ParamField body="attributes" type="object" required>
      Password reset request attributes
      
      <Expandable title="properties">
        <ParamField body="email" type="string" required>
          Email address of the user requesting password reset
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

### Response

#### Success Response (204 No Content)

Returns `204 No Content` regardless of whether the email exists or not. This prevents email enumeration attacks.

<Info>
For security reasons, this endpoint always returns success even if the email doesn't exist in the system. This prevents attackers from using this endpoint to discover valid email addresses.
</Info>

### Examples

<CodeGroup>
```bash cURL
curl -X POST https://api.example.com/api/json/password-reset/request \
  -H "Content-Type: application/vnd.api+json" \
  -d '{
    "data": {
      "type": "password_reset",
      "attributes": {
        "email": "user@example.com"
      }
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/json/password-reset/request', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/vnd.api+json',
  },
  body: JSON.stringify({
    data: {
      type: 'password_reset',
      attributes: {
        email: 'user@example.com'
      }
    }
  })
});

if (response.status === 204) {
  console.log('Password reset email sent (if email exists)');
}
```

```python Python
import requests

response = requests.post(
    'https://api.example.com/api/json/password-reset/request',
    headers={'Content-Type': 'application/vnd.api+json'},
    json={
        'data': {
            'type': 'password_reset',
            'attributes': {
                'email': 'user@example.com'
            }
        }
    }
)

if response.status_code == 204:
    print('Password reset email sent (if email exists)')
```
</CodeGroup>

## Reset Password with Token

### Endpoint

```
POST /api/json/password-reset/reset
```

Complete the password reset using the token received via email.

### Request

#### Headers

<ParamField header="Content-Type" type="string" required>
  Must be `application/vnd.api+json`
</ParamField>

#### Body Parameters

<ParamField body="data" type="object" required>
  The request data object
  
  <Expandable title="properties">
    <ParamField body="type" type="string" required>
      Must be `"password_reset"`
    </ParamField>
    
    <ParamField body="attributes" type="object" required>
      Password reset attributes
      
      <Expandable title="properties">
        <ParamField body="reset_token" type="string" required>
          The reset token received via email
        </ParamField>
        
        <ParamField body="password" type="string" required>
          New password (minimum 8 characters)
        </ParamField>
        
        <ParamField body="password_confirmation" type="string" required>
          Password confirmation (must match password)
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

### Response

#### Success Response (200 OK)

<ResponseField name="data" type="object">
  The user resource with updated password
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      User identifier (UUID)
    </ResponseField>
    
    <ResponseField name="type" type="string">
      Resource type: `"user"`
    </ResponseField>
    
    <ResponseField name="attributes" type="object">
      User attributes
      
      <Expandable title="properties">
        <ResponseField name="email" type="string">
          User's email address
        </ResponseField>
        
        <ResponseField name="confirmed_at" type="string">
          Email confirmation timestamp (auto-confirmed after reset)
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="meta" type="object">
      Response metadata
      
      <Expandable title="properties">
        <ResponseField name="token" type="string">
          New JWT authentication token
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Password Reset Flow

<Steps>
  <Step title="User requests password reset">
    User enters their email address on the password reset request page. Your application sends a POST request to `/api/json/password-reset/request`.
  </Step>
  
  <Step title="System sends reset email">
    If the email exists, the system sends an email with a secure reset token. The email is sent using the configured Swoosh adapter.
  </Step>
  
  <Step title="User receives email">
    User receives an email with a link containing the reset token (e.g., `https://yourapp.com/reset-password?token=abc123...`).
  </Step>
  
  <Step title="User submits new password">
    User clicks the link, enters a new password, and submits the form. Your application sends a POST request to `/api/json/password-reset/reset` with the token and new password.
  </Step>
  
  <Step title="Password updated and user signed in">
    The API validates the token, updates the password, auto-confirms the email (if not already confirmed), and returns a new JWT token. All existing tokens are revoked.
  </Step>
</Steps>

## Security Features

<AccordionGroup>
  <Accordion title="Token expiration">
    Reset tokens have a limited lifetime and expire after a configured period (typically 24 hours). Expired tokens cannot be used to reset passwords.
  </Accordion>
  
  <Accordion title="Single-use tokens">
    Reset tokens can only be used once. After a successful password reset, the token is immediately invalidated.
  </Accordion>
  
  <Accordion title="Email enumeration protection">
    The request endpoint always returns `204 No Content`, preventing attackers from discovering which email addresses are registered.
  </Accordion>
  
  <Accordion title="All tokens revoked">
    When a password is reset, all existing authentication tokens for that user are revoked. This ensures that if an attacker had access to old tokens, they become useless.
  </Accordion>
  
  <Accordion title="Auto-confirmation">
    Users who reset their password have their email automatically confirmed. This is configured in the User resource:
    
    ```elixir
    # From lib/ash_coffee/accounts/user.ex:21
    auto_confirm_actions [:sign_in_with_magic_link, :reset_password_with_token]
    ```
  </Accordion>
</AccordionGroup>


## Email Configuration

The password reset email is sent using the sender configured in the User resource:

```elixir
# From lib/ash_coffee/accounts/user.ex:39-44
resettable do
  sender AshCoffee.Accounts.User.Senders.SendPasswordResetEmail
  password_reset_action_name :reset_password_with_token
  request_password_reset_action_name :request_password_reset_token
end
```

In development, emails are sent to the local mailbox preview at `http://localhost:4000/dev/mailbox`.
